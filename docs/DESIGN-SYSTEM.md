# Halcyon — Design System (DESIGN-SYSTEM.md)

Source of truth for Halcyon's visual + interaction design. Generated by /design-consultation on 2026-06-14. Scope: the wedge surfaces (free reference-match tool / funnel, develop editor, batch + export). Downstream `/plan-design-review` and `/design-review` read this file.

## North star
**"I saw exactly what it did."** Every decision serves the transparency moment: the reference match resolves into *real, editable adjustments the user can watch happen.* If a choice doesn't make the match more legible, tunable, or trustworthy, it doesn't earn its place.

## First principle
**The chrome is grayscale so the photo is the only color in the room.** All UI surfaces are neutral dark. The single warm accent (gold) is a *state signal*, not decoration — it marks what is live (the control being scrubbed), the primary action, and focus. Gold everywhere = gold means nothing.

---

## 1. Color tokens

Neutral grays only (kept near R=G=B; honor these exact values, keep any additions strictly neutral). Dark UI leans on borders + faint shadow, not elevation glow.

```css
:root {
  /* Backgrounds */
  --bg-sunken:    #0A0A0B;  /* canvas letterbox, deepest */
  --bg-base:      #0E0E0F;  /* app background */
  --surface-1:    #161618;  /* panels, top bar */
  --surface-2:    #1C1C1F;  /* raised: cards, menus, tray items */
  --surface-3:    #232327;  /* hover / active rows */

  /* Borders */
  --border-subtle:  #1F1F22;
  --border-default: #262628;
  --border-strong:  #34343A;  /* dividers + slider tracks that must read */

  /* Text */
  --text-primary:   #EDEDEC;  /* headings, live values */
  --text-secondary: #B4B4BA;  /* resting numeric readouts, labels */
  --text-muted:     #8A8A8F;  /* secondary labels, hints */
  --text-faint:     #5C5C62;  /* disabled, placeholders */

  /* Accent — warm gold/brass, state signal only */
  --accent:         #C9A961;
  --accent-hover:   #D8BA74;
  --accent-press:   #B7984F;
  --accent-subtle:  rgba(201,169,97,0.12); /* focus bg, selected tint */

  /* Semantic — desaturated so they never compete with the photo */
  --success: #6F9F74;  /* muted sage */
  --warning: #BFA15A;  /* warm, near-accent */
  --danger:  #C16F6F;  /* muted, never pure red */
  /* No blue "info" color — use --text-secondary. Blue would tint perception. */

  /* Shadow (sparingly) */
  --shadow-1: 0 1px 2px rgba(0,0,0,0.40);
  --shadow-2: 0 4px 16px rgba(0,0,0,0.50);  /* popovers/menus */
}
```

**Color roles:** active control → `--accent`; primary action → `--accent` outline/fill; focus ring → `--accent` + `--accent-subtle`; selected → `--accent-subtle` bg + `--accent` 2px left-border; everything resting → grayscale.

## 2. Typography

UI: **Inter** (or Geist) with `font-feature-settings: "tnum" 1, "cv05" 1`. **All numeric readouts use tabular numerals** so values never shift width while scrubbing. Funnel/marketing display only: one refined face (e.g. a high-contrast grotesque or editorial serif) — never inside the editor.

```css
--font-ui: "Inter", system-ui, sans-serif;
--font-mono: "Geist Mono", ui-monospace, monospace; /* not used for readouts; readouts are tnum UI font */
--font-display: "Hedvig Letters Serif", Georgia, serif; /* funnel only */
```

Scale (size / line-height / tracking / use):
| Token | px / lh | Use |
|---|---|---|
| text-2xs | 11 / 14, +0.02em | slider unit suffixes, micro-labels |
| text-xs | 12 / 16 | numeric readouts, secondary labels |
| text-sm | 13 / 18 | default UI text, panel control labels |
| text-base | 14 / 20 | body, dialog text |
| text-md | 16 / 24, -0.01em | section headers |
| text-lg | 20 / 28, -0.01em | panel titles, empty-state headline |
| text-xl | 28 / 34, -0.02em | onboarding headline |
| display | 48–72 / 1.05, -0.03em | funnel hero only |

Section headers in the develop panel: text-sm, `--text-muted`, uppercase, +0.06em tracking.

## 4. Motion

```css
--dur-fast:120ms; --dur-base:160ms; --dur-slow:240ms;
--dur-signature:400ms;          /* the match-fill, see §6 */
--ease-out: cubic-bezier(0.2, 0, 0, 1);   /* no bounce, ever */
```
Rule: edits feel instant — UI transitions 120–160ms. The **only** longer beat is the signature match-fill. No spring/bounce anywhere.

## 5. Components

**Slider** (the heart of the tool):
- Track 2px, `--border-strong`. Bipolar controls (Exposure, Temp, Tint, Contrast, Whites, Blacks) fill **from center**; unipolar (Saturation, Vibrance) fill from left.
- Resting fill: `--text-secondary`. **Active/scrubbing fill: `--accent`.** Gold appears only while live.
- Thumb 12px circle, `--surface-2` + `--border-strong`; hover 14px; scrubbing → 1px breathing `--accent` focus ring.
- Value readout: right-aligned, tabular, `--text-secondary` at rest → `--text-primary` while scrubbing. Unit suffix in text-2xs `--text-muted`.
- **Double-click resets to default** (120ms ease). Drag on the track, the thumb, or the number (fine-tune). Arrow keys nudge; Shift+arrow ×10.

**Button:**
- Primary (Export): `--accent` 1px outline + `--accent` text on transparent; hover fills `--accent-subtle`; press `--accent-press`. (Outline, not flooded fill — keeps gold restrained.)
- Secondary: `--surface-2`, `--border-default`, `--text-primary`; hover `--surface-3`.
- Ghost/icon: transparent; hover `--surface-2`.

**Panel section:** header row (uppercase muted label + chevron), collapsible, 160ms height ease. Control row = label (text-sm) | slider | value.

**Reference tray (left rail):** vertical filmstrip of 48px reference thumbs; active ref has 2px `--accent` left-border; "+ add reference" dashed tile. Below: **blended-target swatch strip** (the averaged mood-board look) so "match to a board" is legible.

**Match-strength ring:** 2px arc around the 56px reference thumb; track `--border-default`, fill `--accent`, sweeps 0–100%; label below "92% match" (tabular). <60% → small "tune to finish" chip in `--text-muted`. Never red, never a grade.

**Filmstrip (batch):** 56px thumbs, current has `--accent` underline; per-image tiny match-strength dot; processing shows a neutral determinate bar.

**Canvas + before/after:** `--bg-sunken` letterbox. Spacebar-hold = instant before/after swap. Plus a draggable **split handle** (1px `--accent` line, 16px hit area) for side-by-side wipe.

---

## 6. The signature interaction (creative risk #1 — the moat made felt)

On **Apply match**, sliders animate from 0 to their fitted values, staggered 30ms per control, `--dur-signature` (400ms), `--ease-out`. The user *watches the look become editable adjustments.* This is the one place motion exceeds 160ms — it earns it because it is the product's reason to exist. Respects `prefers-reduced-motion` (snaps instantly, no stagger). Skippable (click anywhere). Doubles as onboarding beat 2.

---

## 7. Creative risks (adopted)
1. **Slider fill-in on match** — §6. The signature, the shareable moment.
2. **Match strength as a ring, not a grade** — §5. Confidence, not a test score.
3. **Mood-board tray that visibly resolves N refs into one blended target** — §5. Makes the multi-reference story legible.

## 8. States (every wedge surface)

**Empty (tool first load):** centered dropzone — "Drop a reference look, then your photos." Ghosted before/after example behind it. Starts the contextual onboarding.

**Loading:** neutral skeleton of panels (no colored shimmer). Batch shows a determinate per-image progress bar, count "12 / 40."

**Error:** inline card, never a modal wall. Decode fail → "Couldn't read that file. Supported: JPEG, PNG, TIFF" + Retry. Engine fail → "Match couldn't run — try a different reference" + the photo stays untouched (non-destructive).

**Onboarding (contextual — the hero, NOT a modal wall):** 3 coachmarks tied to real UI, dismissible, remembered in localStorage:
1. arrow to tray — "Drop a reference look."
2. after match (during the fill-in) — "Watch — the look just became real adjustments."
3. "Tune anything, then export." Done.

---

## 9. Accessibility
- Focus rings always visible: 2px `--accent` + 2px offset on dark.
- Full keyboard nav: Tab order, arrow-key slider nudge, `[`/`]` prev/next image, `Space` before/after, `Esc` closes overlays.
- Contrast: text-primary on bg ≈ 14:1; text-muted on surface-1 ≈ 4.6:1 (AA); gold used as text only at ≥16px (≈8:1 on bg).
- `prefers-reduced-motion`: signature fill-in snaps; all transitions ≤ 0 where required.
- Hit areas ≥ 24px even where the visual is 2px (slider track, split handle).

---

## 10. Anti-slop — what Halcyon deliberately rejects
No blue-tinted darks. No purple gradients. No glassmorphism blur over the photo. No neon. No full-width gold or gold gradients. No drop-shadow soup. No bounce/spring. No jittering numbers (tabular only). No centered-everything marketing template. No decorative blobs or 3-column icon grids on the funnel — the working tool *is* the hero.
